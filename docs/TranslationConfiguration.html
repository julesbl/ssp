SSP Translation configuration

To configure the translation routines for just changing various texts or adding another language, currently just french.

  1. Change the name of the name of the translate.change directory to translate.
  2. In config.php set the $lang property to your default language and set $translate to true.

Adding a language

  1. Make copies of the following files in the translate directory with the english code replaced with the new language code, see language codes explained here and list of codes here.
    1. lang_(lang code).conf.php - configure menu entry, language code and reading direction.
    2. lang_(lang code).php - Translations for SSP_datacheck and form error messages.
    3. lang_(lang code)_login.php - Translation for login and allied routines error messages.
    4. lang_(lang code)_admin.php - Translation for admin area of site, menu entries, error mesages etc.
  2. Make a new directory named as the correct language code in templates, copy all the templates from the en directory into it and translate them.
  3. Once the translation routines are enabled the new language will show in the selection menu and will be picked up by the autodetection routines.
